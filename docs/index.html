Why this matters

Translated benchmarks are not enough for clinical language.

Terminology is local

Clinical phrasing, abbreviations, exam conventions, and answer expectations shift by country and by specialty. A model that looks strong in English or translated QA can still fail in native practice.

Evaluation should mirror use

The benchmark centers real medical training material first, then uses translated anchors only for calibration and comparison, not for the headline leaderboard.

Open infrastructure matters

Every dataset should come with provenance, a reconstruction path, and a stable scoring interface so new languages can be added without rewriting the project each time.

Framework design

One benchmark spine, multiple evidence shapes.

Record formats

The project now handles both multiple-choice records and QA-style records, so native clinical datasets do not have to be distorted to fit the same schema.

Build pipeline

Dataset creation scripts can be run through a shared CLI, making local reconstruction and future GitHub Actions benchmarking straightforward.

Scoring logic

MCQ datasets use accuracy. QA datasets use exact match with token-level F1. Native-only task weighting remains the main public aggregation.

Contributor workflow

Adding a new country should feel procedural, not heroic.

1

Find a real local source

Prefer official exams, public archives, or clinically reviewed educational sets with stable answer keys.

2

Choose the right record format

Use MCQ for answer-option data, or QA formats for span or generative answers when that is the true source shape.

3

Ship provenance with code

Every dataset should come with a reconstruction script, access mode, license note, and documentation page.

4

Keep leaderboards honest

Native exam datasets should drive the flagship score. Translated or differently formatted datasets can still be reported clearly.

What this project can become

A EuroEval-style benchmark for medicine, built from Europe’s own training and clinical language.

The point is not to create one more translated leaderboard. The point is to build the open benchmark infrastructure that every European medical language still lacks.

EuroMedEval is designed as an open-source benchmark framework with a public GitHub Pages front door.
